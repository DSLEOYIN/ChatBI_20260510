# 右侧数据可视化讨论

## 背景

用户反馈 `index_v2.html` 原型存在问题：
- 右侧看板是固定的 4个KPI + 2个图 + 1个表
- 无论左边问什么问题，右侧结构完全不变
- 用户问题可能是闲聊、搜索、数据分析等多种类型，固定模板不适用

## 用户需求

用户要求右侧看板应该是**动态生成的**，根据左侧问题的意图自动适配：
- 闲聊时 → 右侧不需要显示内容
- 简单查询 → 只需要显示单个数值
- 复杂分析 → 需要完整的图表组合
- 搜索 → 显示搜索结果卡片
- 指标定义 → 显示定义解释卡片

## 讨论的解决方案

### 方案一：意图驱动 + 动态卡片系统（最终选择）

核心思路：右侧是左侧的"答案延伸"，AI 根据问题意图动态生成最合适的可视化卡片。

#### 卡片类型系统

| 卡片类型 | 触发场景 |
|---------|---------|
| KPI_SINGLE | 简单查询（如"GS8库存多少"）|
| KPI_GRID | 分析问题（多个指标对比）|
| CHART_BAR / CHART_LINE | 趋势/对比分析 |
| TABLE | 明细数据展示 |
| CONCLUSION | AI 分析结论和建议 |
| ALERT | 预警信息 |
| SEARCH | 联网搜索结果 |
| DEFINITION | 指标定义解释 |

#### 意图识别逻辑

```javascript
detectIntent(userInput) → IntentType
- CHAT: 闲聊类（你好、早上好等）→ 不生成卡片
- SIMPLE_QUERY: 含"多少"但不含"分析" → 单KPI卡
- ANALYSIS: 含"分析、诊断、查看" → 完整卡片组合
- COMPARISON: 含"对比、比较" → 图表对比
- SEARCH: 含"搜索、查询" → 搜索结果卡
- DEFINITION: 含"什么是、定义" → 定义卡
- ALERT: 含"预警、异常、风险" → 预警卡
```

#### 三种展示状态

```
状态1：纯闲聊/搜索 → 右侧完全折叠，100%对话空间
状态2：简单数据问题 → 右侧展开KPI单卡 + 文字结论
状态3：复杂分析问题 → 右侧展开多卡片组合
```

### 方案二：视图模式切换（备选）

在右上角加视图切换器，类 PowerBI/Tableau 书签功能。但用户认为场景太多不适用。

### 方案三：动态卡片超市（长期方向）

最灵活但开发量大。

## 关键代码片段

### 意图识别

```javascript
function detectIntent(userInput) {
    const input = userInput.toLowerCase();

    if (/^(你好|您好|早上好|下午好|晚上好|嗨)/.test(input)) return IntentType.CHAT;
    if (/什么是|定义|解释/.test(input)) return IntentType.DEFINITION;
    if (/搜索|查找|查询/.test(input)) return IntentType.SEARCH;
    if (/预警|警告|异常|问题|风险/.test(input)) return IntentType.ALERT;
    if (/对比|比较|差异/.test(input)) return IntentType.COMPARISON;
    if (/(多少|几个)/.test(input) && !/分析|诊断|对比/.test(input)) return IntentType.SIMPLE_QUERY;
    if (/分析|诊断|查看/.test(input)) return IntentType.ANALYSIS;

    return IntentType.SIMPLE_QUERY;
}
```

### 卡片生成

```javascript
function generateCardsByIntent(intent, userInput, aiResponse) {
    const cards = [];
    const inputLower = userInput.toLowerCase();
    const input = userInput;

    switch (intent) {
        case IntentType.CHAT:
            return cards; // 无卡片
        case IntentType.SIMPLE_QUERY:
            // 生成单KPI卡
            if (inputLower.includes('gs8')) {
                cards.push({ type: CardType.KPI_SINGLE, ... });
            }
            return cards;
        case IntentType.ANALYSIS:
            // 生成完整卡片组合
            cards.push({ type: CardType.KPI_GRID, ... });
            cards.push({ type: CardType.CHART_LINE, ... });
            cards.push({ type: CardType.CHART_BAR, ... });
            cards.push({ type: CardType.TABLE, ... });
            cards.push({ type: CardType.CONCLUSION, ... });
            return cards;
        // ... 其他意图
    }
}
```

### 卡片渲染

```javascript
function renderCards(cards) {
    const container = document.getElementById('cards-container');
    const emptyState = document.getElementById('cards-empty-state');

    if (!cards || cards.length === 0) {
        container.innerHTML = '';
        emptyState.classList.remove('hidden');
        return;
    }

    emptyState.classList.add('hidden');
    currentCards = cards;

    const cardsHtml = cards.map((card, index) => renderCard(card, index)).join('');
    container.innerHTML = cardsHtml;

    // 渲染图表
    cards.forEach((card, index) => {
        if (card.type === CardType.CHART_BAR || card.type === CardType.CHART_LINE) {
            const chartEl = container.querySelectorAll('.chart-card-dynamic')[index];
            if (chartEl) {
                renderChart(chartEl.querySelector('.chart-container').id, card.data.chartType);
            }
        }
    });

    addToCardPool(cards);
}
```

## 结论

动态卡片系统的设计思路是正确的，但实现过程中遇到了未解决的 bug。需要进一步调试 `renderCards` 函数和 DOM 操作，确认卡片是否正确生成并插入到页面中。

右侧看板应该显示的内容取决于左侧问题的意图：
- 闲聊 → 不显示卡片
- 简单查询 → 单个 KPI 卡
- 分析 → KPI网格 + 图表 + 表格 + 结论
