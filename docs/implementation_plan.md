# ChatBI 交互原型 2.0 重造计划

我们旨在将原有的轻量级悬浮聊天机器人，重构为一个**面向企业级用户的 AI 数据决策中心（ChatBI Workspace v2.0）**。新原型将以极其精致的科技感视觉风格（玻璃摩登、暗黑流光、无缝动效），完美演示从“用户自然语言输入”到“SQL 自动生成”、“数据图表联动”以及“深度洞察报告”的完整闭环。

---

## 核心设计理念与头脑风暴构想

### 1. 突破“聊天框”的形态限制
- **原问题**：BI 用户需要看大宽表、看复杂对比图，而 440px 宽的浮窗极难承载这些信息。
- **重构方案**：保留**右下角精致悬浮球（Launcher）**，但点击后支持**双态无缝切换**：
  - **精简态（Floating Panel）**：轻量问答，快速查询某个指标。
  - **工作区态（Split Workspace）**：一键扩展为全屏，左侧为**深度对话与思考链**，右侧为**可视化交互画布（包含 ECharts 图表、多维数据表、数据字典）**。

### 2. 深度思考的“透明化”与“确定性”
- **原问题**：用户不知道 AI 在后台干了什么，对生成的 SQL 和图表缺乏信任度。
- **重构方案**：引入可视化的 **Chain of Thought (CoT) 节点式流程图**。对于“深度思考”模式，展开显示：
  1. `意图识别与消解` -> 2. `知识检索 (Dify)` -> 3. `SQL 智能生成 (带高亮和一键复制)` -> 4. `图表配置生成` -> 5. `深度洞察提炼`。
  用户能够清晰地点击并查看每一步的运行日志和原始 SQL。

### 3. 数据呈现的“生动性”与“交互性”
- **原问题**：原原型中的销量和预警全为纯文本，无法带来 BI 的冲击感。
- **重构方案**：
  - 引入 **ECharts 库**（通过内网兼容的静态引入/CDN），在页面上渲染**真实可交互的图表**（支持 Hover Tooltip、图例切换、缩放）。
  - 设计**精细的数据报表（Interactive Data Table）**，支持表头排序、搜索和一键“Pin 到看板”模拟。
  - 添加**指标警告卡片（Alert Badges）**，库存天数（周转率）高风险时呈红黄闪烁动效。

### 4. 消除“冰冷感”的高端设计美学
- **主题配色**：采用**广汽国际（GAC）**的专属深邃碳灰（Carbon Dark, `#0B0F19`）为底色，融入科技感电极蓝（Electric Blue）与极光绿（Teal-Green），配合微妙的流光渐变（Smooth Gradients）和微毛玻璃（Glassmorphism）质感。
- **微交互**：打字机效果、气泡浮动、流程线动态连结、按钮悬浮的微物理回弹。

## 用户评审要点

> [!IMPORTANT]
> **关于技术依赖的评审**：
> 1. 原型中为了展现 **ECharts**，我们需要引入 ECharts 的 CDN 资源（如 `cdnjs` 或 `bootcdn`）。若在绝对无网的局域网环境测试，我们将准备 SVG/Canvas 备用渲染层。
> 2. 我们将新建 `index_v2.html`，保留原有的 `index_robot_only.html` 以做版本对照，不破坏原有资产。

## 验证与演示计划

### 自动化与手动测试
1. **浏览器直接访问验证**：在浏览器中打开 `index_v2.html`。
2. **模拟提问流验证**：
   - 输入“本月销量”，观察思考链以卡片形式逐步展开，最后渲染出销量走势图表。
   - 输入“库存预警”，观察红色预警指标和多维周转数据表。
3. **尺寸缩放测试**：拖动浏览器窗口，测试从 PC 宽屏、平板分屏到手机窄屏的完全自适应性。
4. **快捷键响应测试**：在任意聚焦状态下按下快捷键，验证功能的即时唤醒。
