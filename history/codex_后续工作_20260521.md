## 1. 今日接续背景

当前项目仍处于 ChatBI Workspace 2.0 第一阶段：以前端原型 `前端原型/index_v3_dynamic.html` 为唯一视觉与交互基准，先通过 React + FastAPI mock 跑通完整业务链路，再逐步替换为真实 MySQL、Dify、Tavily MCP、LangGraph Agent 与权限体系。

本阶段核心目标不是重新设计 UI，而是把原型中已经确认的体验完整迁移到 React 项目：

1. 悬浮球、精简窗口、全屏工作区、历史会话抽屉必须与原型对齐。
2. 左侧展示用户问题、Agent 可理解执行链、SQL/工具复核信息；最终业务答案统一在右侧业务画布输出。
3. 右侧画布由结构化 payload 驱动，支持最终回答、KPI、风险预警、图表、表格、结论建议、搜索结果、指标定义等组件。
4. 第一阶段后端接口全部 mock，重点保证会话、SSE 流式问答、执行链、画布、下载、数据资产、Skill 注册等流程稳定。
5. 用户可见的是“可理解的执行链/审计链”，不是完整内部推理。

当前本机验证端口：

- 前端：`http://127.0.0.1:5174/`
- 后端：`http://127.0.0.1:8001/`
- 前端环境变量：`VITE_API_URL=http://127.0.0.1:8001/api`

说明：`5173` 端口当前被其他项目占用，因此本轮验证临时使用 `5174`。
